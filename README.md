# Function-Calling
Function calling на практике: модель получает обычный текстовый запрос, сама выбирает нужную функцию, подставляет аргументы, а дальше код уже выполняет реальное действие.


Сейчас в ноутбуке есть два сценария:

- отправить сообщение пользователю в Telegram;
- получить события из Google Calendar за выбранный период.

Идея: пользователь пишет что-то вроде «напиши пользователю ...» или «какие события в календаре с такой-то даты», а модель решает, какую функцию нужно вызвать.

## Ноутбук разделён на несколько частей:

1. **Telegram-функция**  
   Загружает `chat_id` пользователя из локального файла и отправляет сообщение через Telegram Bot API.

2. **Google Calendar-функция**  
   Авторизуется через Google OAuth и забирает события из основного календаря за нужный промежуток дат.

3. **Схема tools/functions**  
   Описывает функции в формате, понятном модели: названия, параметры и описания.

4. **Запрос к модели**  
   Модель получает пользовательский текст и возвращает `tool_calls`, если считает, что нужно вызвать одну из функций.

5. **Реальное выполнение функции**  
   Код разбирает ответ модели, вызывает нужную Python-функцию и передаёт результат обратно в модель, чтобы получить нормальный финальный ответ.


## Переменные окружения

Для работы нужен файл `.env` в корне проекта:

```env
TELEGRAM_BOT_API_KEY=your_telegram_bot_token
OPENROUTER7_API_KEY=your_openrouter_api_key
```

## Файлы, которые нужны локально

### `tg_users.txt`

В этом файле хранится соответствие username -> chat_id для Telegram.

Пример:

```json
{
  "Username": "123456789"
}
```

Функция `send_telegram_msg` берёт username, ищет по нему chat_id и отправляет сообщение.

### `credentials.json`

Этот файл нужен для Google Calendar API. Его можно получить в Google Cloud Console после создания OAuth Client.

При первом запуске Google попросит авторизоваться в браузере, а потом создастся файл `token.json`.

### `token.json`

Создаётся автоматически после первой авторизации в Google. 

## Как запустить

1. Склонировать репозиторий.
2. Установить зависимости.
3. Создать `.env` с ключами.
4. Добавить `tg_users.txt`.
5. Положить рядом `credentials.json` для Google Calendar.
6. Открыть ноутбук `Function Calling.ipynb`.
7. Запустить ячейки сверху вниз.
